# 互动电影游戏辅助开发软件设计文档

## 1. 项目概述
本项目旨在开发一款互动电影游戏的辅助开发软件，允许用户通过可视化界面设计、编辑和预览互动电影流程。用户可以在平面画布上管理视频节点、交互逻辑和游戏数值，并进行即时试玩。

## 2. 核心功能

### 2.1 画布编辑系统
用户在一个平面画布上进行可视化操作，主要包含以下功能：
- **节点管理**：
  - 添加节点
  - 移动节点位置
  - 编辑节点属性（视频源、名称、是否 Start 节点）
  - **Start 节点限制**：全局仅允许一个 Start 节点，设置新 Start 节点时会自动询问是否替换旧节点。
- **连接管理**：
  - 添加连接关系（拖拽创建）
  - 修改连接属性（条件判断、数值变更）
  - 删除连接关系（选中后点击删除按钮）
  - **连接约束**：节点左侧 Handle 仅作为输入，不可作为输出源。
- **操作管理**：
  - 在节点上添加交互“操作” (Action)
  - 修改操作属性（类型、时间、样式、位置）
  - 删除操作

### 2.2 数值系统管理
- 为整个系统定义“数值”类型（变量）。
- 支持添加、删除、重命名数值定义。
- 支持设置数值的默认值（布尔值、数字、字符串）。
- 用于控制游戏逻辑分支和状态记录。

### 2.3 试玩/预览系统
- 根据编辑好的节点和逻辑进行模拟运行。
- **视频播放**：
  - 支持本地视频文件播放（自动处理 `file://` 协议）。
  - 支持竖屏视频适配 (`object-contain`)。
  - 节点悬停预览：鼠标悬停在节点视频区域时自动播放预览。
- **交互逻辑**：
  - **按钮交互**：在指定时间或视频结束时显示选项按钮。
  - **自动跳转**：视频播放结束后自动跳转到下一节点（无 UI 交互）。
  - **样式自定义**：支持自定义按钮颜色、文字颜色和位置（百分比坐标）。
- **状态管理**：
  - 实时追踪变量变化。
  - 独立运行环境，不影响编辑数据。

### 2.4 脚本导入与生成系统
- **脚本解析**：系统支持解析特定格式的剧本/脚本文件。
- **自动生成流程图**：
  - 根据脚本内容自动创建对应的“节点”。
  - 根据脚本中的选项分支自动创建“操作”和“连接关系”。
  - 自动布局生成的节点位置。
- **后续配置**：用户在生成的基础上，只需补充关联具体的“视频对象”以及细化“数值判断”和“数据变更”逻辑即可。

### 2.5 辅助功能
- **单实例运行**：应用限制为单实例运行，避免多开冲突。
- **测试模板**：支持保存当前项目为测试模板，或快速加载默认测试用例。

## 3. 数据结构与属性定义

### 3.1 节点 (Node)
代表游戏中的一个视频片段或场景。
- **ID**：唯一标识符。
- **Label**：节点名称。
- **Video Source**：关联的视频文件绝对路径。
- **Is Start Node**：标识是否为游戏入口（全局唯一）。
- **Actions**：包含的操作列表。

### 3.2 操作 (Action)
代表用户在特定时间点可以进行的交互（如点击选项）或自动逻辑。
- **ID**：唯一标识符。
- **Label**：显示文本或标识名。
- **Type**：
  - `default`: 按钮交互（需点击）。
  - `auto`: 自动跳转（视频结束自动触发）。
- **Time Mode**：
  - `onEnd`: 视频播放结束时触发/显示。
  - `custom`: 在指定时间段内显示（仅 `default` 类型）。
- **Time Range** (`startTime`, `endTime`): 自定义显示时间段。
- **Position** (`x`, `y`): 按钮在屏幕上的百分比坐标（仅 `custom` 时间模式）。
- **Style**:
  - `color`: 按钮背景色。
  - `textColor`: 按钮文字颜色。
- **Condition**: 显示该操作的前置条件（变量判断）。

### 3.3 连接关系 (Connection)
定义从一个操作触发后跳转到下一个状态的逻辑。
- **Source**：源节点 ID。
- **Source Handle**：源 Action ID。
- **Target**：目标节点 ID。
- **Conditions**：触发跳转的前置条件列表（与的关系）。
- **Mutations**：跳转时执行的变量变更列表。

### 3.4 数值类别 (Global Value/Variable)
定义游戏中使用的全局变量。
- **ID**：唯一标识符。
- **Name**：变量名。
- **Type**：数据类型 (`boolean`, `number`, `string`)。
- **Default Value**：初始值。
- **Is Persistent**：是否跨局保存（存档）。

## 4. 试玩逻辑流程

1. **初始化**：
   - 加载所有数值类别的初始值。
   - 找到标记为 `isStartNode` 的节点作为当前节点。
   - 打开 `PlayerWindow`，初始化播放器状态。

2. **视频播放**：
   - 播放当前节点的视频。
   - `VideoPlayer` 实时更新 `currentTime` 和 `isEnded` 状态到 `PlayerWindow`。

3. **交互检测**：
   - `Overlay` 组件监听 `PlayerWindow` 的状态。
   - **Auto Action**: 若当前节点有 `auto` 类型 Action 且视频结束 (`isEnded=true`)，且满足条件，自动触发跳转。
   - **Default Action**:
     - 若 `timeMode='onEnd'` 且 `isEnded=true`，显示按钮。
     - 若 `timeMode='custom'` 且 `currentTime` 在范围内，显示按钮（按自定义位置和样式）。

4. **用户触发**：
   - 用户点击显示的按钮。
   - 系统查找该 Action 对应的连接线 (`Edge`)。

5. **逻辑执行与跳转**：
   - 执行连接线上的 `Conditions` 检查。
   - 执行 `Mutations` 更新变量。
   - 将 `CurrentNode` 更新为连接线的 `Target` 节点。
   - 重置播放器状态 (`isEnded=false`, `currentTime=0`)，加载新视频。

### Phase 7: 功能增强 (2026-03-01)
- **Electron 单实例**：限制应用单开，唤起已打开窗口。
- **视频预览**：节点内视频自动截取第 3 秒帧，悬停播放预览。
- **竖屏适配**：播放器支持 `object-contain` 模式。
- **变量编辑**：支持重命名变量和修改默认值。
- **自定义位置**：Action 支持百分比坐标定位。
- **模板保存**：支持保存当前项目为测试模板。
- **节点尺寸**：固定节点视频预览尺寸 (200x112) 防止跳变。

### Phase 8: 自动跳转与播放器重构 (2026-03-01)
- **Auto Action**：新增 `auto` 类型 Action，视频结束后自动跳转，无按钮显示。
- **视觉区分**：编辑器中 Auto Action 显示黄色闪电图标。
- **样式支持**：Overlay 按钮支持自定义背景色和文字颜色。
- **播放器重构**：将 `currentTime` 和 `isEnded` 状态提升至 `PlayerWindow`，解决 DOM 事件监听不可靠导致的 Overlay 不显示问题。
- **Z-Index 修复**：提高 Overlay 层级，防止被视频遮挡。

### Phase 9: 交互细节优化 (2026-03-01)
- **连接线删除**：Connection Panel 增加删除按钮，显示连接详情。
- **Start 节点约束**：设置 Start 节点时自动检测并询问替换旧节点。
- **Handle 样式**：Auto Action 的连接点显示为黄色，区分于普通 Action。
- **输入点约束**：节点左侧输入点限制为仅接收连接 (`isConnectableStart={false}`)。
