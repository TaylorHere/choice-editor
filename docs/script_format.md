# 互动电影脚本格式规范 v1.0

该脚本使用扩展的 Markdown 格式，用于描述互动电影的剧情流程。
解析器将根据此格式自动生成节点图。

## 基本规则
1. **场景 (Scene)**: 使用 `##` 标题定义一个节点（场景）。
2. **标识符 (ID)**: 标题后的 `{#id}` 为可选的节点 ID，若不填则自动生成。
3. **内容 (Content)**: 标题下方的文本为备注或剧本描述。
4. **选项 (Options)**: 使用无序列表 `- [选项文本] -> (目标节点标题或ID)` 定义交互选项。
5. **初始节点**: 使用 `[START]` 标记初始节点。

## 示例

```markdown
## 序章：苏醒 {#prologue} [START]
主角从昏迷中醒来，发现自己身处一个陌生的房间。
周围很暗，只有一扇门透出微光。

- [查看四周] -> (room_inspect)
- [走向门口] -> (door_check)

## 检查房间 {#room_inspect}
房间里只有一张旧床和一台坏掉的收音机。
收音机里传出沙沙的电流声。

- [再次尝试开门] -> (door_check)

## 检查门口 {#door_check}
门被锁住了。但是门把手上挂着一把钥匙。
这太奇怪了。

- [使用钥匙开门] -> (corridor)
- [大声呼救] -> (game_over)

## 走廊 {#corridor}
门打开了，外面是一条长长的走廊。
（此处未完待续）

## 结局：被发现 {#game_over}
你的呼救声引来了看守。
游戏结束。
```

## 解析逻辑
- 每一段 `##` 解析为一个 `StoryNode`。
- 列表项 `- [Label] -> (Target)` 解析为 `StoryAction` 和 `Connection`。
- 如果目标节点不存在，解析器应生成一个占位节点或标记错误。
