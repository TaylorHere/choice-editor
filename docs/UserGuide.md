# Choice Editor - 用户使用手册

## 1. 简介
Choice Editor 是一款专为互动视频创作而设计的可视化编辑器。通过它，你可以将多个视频片段（Scenes）连接起来，设置分支选项（Actions），并添加复杂的逻辑判断（Logic），从而构建出丰富的互动叙事体验。

---

## 2. 快速入门

### 启动应用
1. 打开应用，你会看到一个空白的网格画布（Canvas）。
2. 左侧是你的创作区域，右侧是属性面板（Right Panel）。
3. 点击右侧面板顶部的 **"Add Scene"** 按钮，创建一个新的场景节点。

---

## 3. 核心概念与功能

### 3.1 场景节点 (Scene Node)
场景是互动视频的基本单元。每个节点代表一段视频播放过程。

*   **创建节点**：点击 "Add Scene"。
*   **编辑节点**：点击画布上的节点，右侧面板会显示其属性。
*   **视频源**：在右侧面板的 "Video Source" 输入框中粘贴视频 URL 或本地文件路径。
    *   *提示：本地路径支持拖拽或手动输入，例如 `/Users/name/video.mp4`*。
*   **起始节点**：勾选 "Is Start Node"，该节点将作为故事的起点（带有绿色 START 标记）。

### 3.2 动作与选项 (Actions)
动作决定了用户可以做什么，以及剧情如何发展。

*   **添加动作**：在节点属性面板中，点击 "Add Action"。
*   **动作类型 (Type)**：
    *   **Button (默认)**：在屏幕上显示一个可点击的按钮。
    *   **Auto Jump (自动跳转)**：视频播放结束后，无需点击自动跳转到下一个节点。
*   **显示时机 (Time Settings)**：
    *   **Show on End**：视频播放结束时显示按钮（常用于传统互动剧）。
    *   **Show during Time Range**：在视频播放过程中（例如第 2-5 秒）显示按钮。
        *   *QTE 模式*：配合超时跳转功能使用。
*   **超时跳转 (Timeout Jump)**：
    *   仅对 "Show during Time Range" 有效。
    *   勾选 "Enable Timeout Jump"。
    *   节点右侧会出现一个灰色的连接点。
    *   **功能**：如果时间结束用户仍未点击，则自动跳转到灰色连线指向的节点。
*   **自定义样式**：你可以设置按钮的颜色、文字颜色以及在屏幕上的位置（X/Y 坐标）。

### 3.3 逻辑与变量 (Logic & Variables)
让你的故事不仅仅是简单的分支，而是拥有“记忆”。

*   **变量管理**：点击右侧面板顶部的 "Variables" 标签页。
    *   创建变量（如 `score`, `hasKey`）。
    *   设置初始值。
*   **修改变量 (Mutations)**：
    *   在连线（Edge）或逻辑节点上设置。
    *   例如：点击“捡起钥匙”选项后，执行 `hasKey = true`。
*   **逻辑节点 (Logic Node)**：
    *   点击 "Add Logic" 添加一个菱形节点。
    *   **功能**：不播放视频，仅做逻辑判断。
    *   **分支 (Branches)**：
        *   添加多个分支条件（如 `score > 60` -> 结局A，`Else` -> 结局B）。
        *   引擎会自动评估条件并无缝跳转到对应结果。
    *   **托底机制 (Fallback)**：
        *   Logic 节点会优先匹配所有带有条件的分支。
        *   如果所有条件都不满足，系统会自动寻找一个没有条件的 **Else** 分支作为出口。
        *   如果连 Else 分支也没有，流程将会在该节点终止（Dead End）。

## 4. 操作指南

### 4.1 连线 (Connecting)
*   **建立连接**：从节点右侧的 **小圆点 (Handle)** 拖拽出一条线，连接到另一个节点的左侧。
*   **删除连接**：选中连线，按 `Backspace` 或 `Delete` 键。
*   **连接属性**：点击连线，右侧面板可以设置：
    *   **Condition**：只有满足特定条件（如 `money > 100`）才允许通过此路。
    *   **Mutation**：通过此路时修改变量数值。

### 4.2 预览与测试
*   **视频预览**：鼠标悬停在节点封面上，可以快速预览视频片段。
*   **完整测试**：
    *   点击右侧面板顶部的 **"Load Test" (烧杯图标)**。
    *   这会加载一个内置的演示项目（Attack/Defense 场景）。
    *   点击 **"Play" (播放图标)** 进入全屏互动模式。

### 4.3 项目保存与加载
*   **Save Project**：将当前所有节点、连线和变量保存为 `.json` 文件。
*   **Load Project**：读取之前的 `.json` 文件，恢复工作进度。
*   **Save Template**：将当前项目保存为浏览器缓存的默认模板（刷新页面不丢失）。

## 5. 高级技巧

### QTE (快速反应事件) 制作流程
1.  创建一个节点，上传视频。
2.  添加一个 Action，类型为 Button。
3.  Time Settings 设为 `Custom`，比如 Start: 2000ms, End: 5000ms。
4.  勾选 `Enable Timeout Jump`。
5.  **成功分支**：从该 Action 的彩色 Handle 连线到“成功节点”。
6.  **失败分支**：从该 Action 的灰色 Timeout Handle 连线到“失败节点”。
7.  **效果**：视频播放到 2秒时按钮出现；如果在 5秒前点击，进入成功节点；如果 5秒时未点击，自动进入失败节点。

### 逻辑分支制作流程
1.  创建一个 "Check Score" 的逻辑节点。
2.  在右侧面板添加分支：
    *   分支 1: `score >= 100` (Label: High Score)
    *   分支 2: `Else` (Label: Normal)
3.  从上一个剧情节点连线进入逻辑节点。
4.  从逻辑节点的 "High Score" Handle 连线到“完美结局”。
5.  从 "Normal" Handle 连线到“普通结局”。
6.  **效果**：玩家到达此处时，系统瞬间判断分数并自动导向对应结局，玩家无感知。
